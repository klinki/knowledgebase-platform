# Text Formatting

Use standard Markdown text formatting and keep emphasis restrained.

## Mandatory Rules

- MUST use inline code for identifiers, filenames, commands, and literals.
- MUST keep emphasis functional, not decorative.
- MUST use blockquotes only for quoted or callout-style content.
- MUST avoid excessive horizontal rules.

## Blockquotes

```markdown
> Blockquote
> Multiple lines
> in one quote
```

Use blockquotes for quoted content or lightweight note-style sections. Do not
use them as a general layout mechanism.

## Horizontal Rules

Use horizontal rules sparingly when they add real structure.

## Do

- Prefer inline code over emphasis for technical identifiers.
- Keep formatting consistent within a document.
- Use emphasis to clarify meaning, not to add visual noise.

## Do Not

- Stack multiple emphasis styles unless the meaning requires it.
- Use blockquotes for non-quoted body text.
- Add horizontal rules where a heading or spacing is enough.
